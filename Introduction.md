目前可分析的資料、位置與說明
1.內部營運資料 (Internal Operational Data)
位置： 課堂提供的模擬資料庫 (SQL Server / MySQL) 或 ERP 系統導出的 CSV 檔案。
說明： 這是 BI 分析的最基本層次。我將分析銷售紀錄、庫存周轉率與客戶基本資料。透過清洗這些結構化資料，練習計算 KPI（關鍵績效指標），例如「年度銷售成長率」與「產品毛利貢獻度」，藉此掌握企業內部營運現況。

2.公開政府開放資料 (Government Open Data)
位置： 政府資料開放平臺 或 Kaggle 競賽數據集。
說明： 為增加專案的深度，我會抓取人口統計、消費者物價指數 (CPI) 或地區交通流量等資料。將這些外部數據與內部銷售資料結合，進行「交叉對比分析」，練習判斷外部經濟環境如何影響企業績效。

3.數位社群與網路行為資料 (Web & Social Media Data)
位置： Google Analytics 報表或透過 Python 爬取的社群媒體關鍵字。
說明： 針對非結構化資料進行分析，例如顧客評論情緒分析或網頁點擊流 (Clickstream)。這能幫助我理解消費者的潛在偏好，並在學期專案中展現從「預測分析」到「決策建議」的完整 BI 流程。
